# Conversation Knowledge - Simplified Syntax

## 🎯 简化语法 vs 传统语法

### /load (加载话题)

| 语法 | 示例 | 说明 |
|------|------|------|
| **新语法** (推荐) | `/load AI 智能体分类` | 空格分割，简洁 |
| | `/加载 AI 智能体分类 summary` | 带格式参数 |
| **旧语法** (仍支持) | `/load --topic "AI 智能体分类"` | 传统参数格式 |
| | `/加载 --topic "AI" --format summary` | 完整参数 |

---

### /record (记录对话)

| 语法 | 示例 | 说明 |
|------|------|------|
| **带上下文** | `/加载 AI 智能体` | 先加载话题 |
| | `/记录 --summary "讨论了..."` | 自动使用上下文话题 |
| **手动指定** | `/记录 --topic "AI" --summary "..."` | 完整参数 |
| **强制新建** | `/记录 --topic "AI" --summary "..." --new` | 强制创建新话题 |

---

### /list (列出最近)

| 语法 | 示例 | 说明 |
|------|------|------|
| **新语法** | `/list 30 10` | 位置参数：days, limit |
| **旧语法** | `/list --days 30 --limit 10` | 命名参数 |

---

### /summarize (总结)

| 语法 | 示例 | 说明 |

| | `/总结 所有` | 总结所有 |

## 💡 语法规则

### 新语法特点

1. **空格分割** - 参数用空格分隔，不需要 `--`
2. **位置参数** - 第一个参数通常是话题名
3. **可选格式** - 最后一个词可以是格式（full/summary/timeline）
4. **中文友好** - 完美支持中文话题名

---

### 旧语法特点

1. **命名参数** - 使用 `--topic`, `--format` 等
2. **引号包裹** - 话题名需要引号
3. **兼容性好** - 适合脚本化

---

## 🚀 快速上手

```bash
# 最简单的用法
/加载 话题名
/记录 --summary "摘要"

# 带格式
/加载 话题名 summary
/导出 话题名 markdown

# 完整参数（旧语法）
/加载 --topic "话题名" --format summary
/导出 --topic "话题名" --format markdown
```
